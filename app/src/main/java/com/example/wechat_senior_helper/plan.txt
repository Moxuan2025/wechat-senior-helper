**问题**：进入聊天页后，底部可能是文本输入模式，需先切换到“按住说话”再长按录音。  
**解决**：在 `sendVoiceMessage` 中加入 OCR 检测与自动切换。

---

### 修改文件：`WeChatVoiceFlow.kt`

**新增方法** `ensureVoiceMode()`  
- 截取屏幕（全屏或底部区域） → OCR 识别文本  
- 若不包含“按住说话” → 点击 `(0.1f, 0.99f)` 切换按钮 → 等待 500ms  

**修改 `sendVoiceMessage`**  
- 调用 `ensureVoiceMode()` 后，再执行长按录音。

---

### 核心代码

```kotlin
// WeChatVoiceFlow.kt

private suspend fun ensureVoiceMode() {
    // 1. 截屏（根据你的截图工具调整，这里示例用 AccessibilityService 截图）
    val bitmap = screenshotUtil.captureScreen() ?: return
    // 2. OCR 识别
    val text = mLKitOcrEngine.recognize(bitmap).text
    bitmap.recycle()
    // 3. 判断是否已在语音模式
    if (!text.contains("按住说话")) {
        // 4. 点击切换按钮（坐标已由你实测确定）
        input.clickByRatio(0.1f, 0.99f)
        delay(500) // 等待录音按钮出现
        // 可选：再次检查，但正常一次足够
    }
}

suspend fun sendVoiceMessage(durationMs: Long = 2000L): Boolean {
    return try {
        ensureVoiceMode()                          // 确保底部是录音按钮
        input.longPressByRatio(0.5f, 0.99f, durationMs) // 长按录音
        true
    } catch (e: Exception) {
        Log.e(TAG, "语音发送失败: ${e.message}")
        false
    }
}
```

### 依赖说明
- `screenshotUtil.captureScreen()`：你的项目里全局截图工具，返回 `Bitmap?`。若无，可用 `AccessibilityService` 的 `takeScreenshot`（API 30+）。
- `input.clickByRatio()`：你现有的点击方法。
- `mLKitOcrEngine.recognize(bitmap)`：已接入的 ML Kit OCR，返回 `Text` 对象，`.text` 即为全部识别文字。

---

**效果**：识别到无“按住说话”时自动点击切换，再长按录音，彻底解决点击位置错误的问题。
