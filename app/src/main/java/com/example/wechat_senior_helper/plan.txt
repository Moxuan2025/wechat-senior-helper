### 整合指南：意图识别完全复用现有功能

现有模块已经完全具备**搜索联系人、发送语音、发起通话**的能力，意图识别的执行端只需按顺序调用它们。思路如下：

```
意图解析(关键词/LLM) → 得到 action + target
   ↓
【统一入口】ensureChatWithTarget(target)
   ├─ 不在微信 → 启动微信并等待
   ├─ 已在微信但非目标聊天 → 返回首页 → 调用 WeChatSearchContactFlow.execute(target)
   └─ 已在目标聊天 → 直接跳过
   ↓
根据 action 执行：
   ├─ voice  → WeChatVoiceFlow.sendVoiceMessage()
   ├─ video  → WeChatVideoCallFlow.makeCall(useVoice)
   └─ (可扩展)
```

**关键点：没有任何新造的执行逻辑，全部复用已有 Flow。**

---

## 一、需要微调的代码（确保复用）

### 1. `IntentHandler.executeConfirmed()` 确保调用已有 Flow

```kotlin
// IntentHandler.kt
suspend fun executeConfirmed(pending: PendingAction): String {
    return try {
        // 核心：导航到目标聊天（内部复用搜索 Flow）
        val chatReady = chatNavGuard.ensureChatWithTarget(pending.target)
        if (!chatReady) {
            return "没有找到「${pending.target}」，请检查名字"
        }

        // 执行操作（直接复用已有 Flow）
        when (pending.action) {
            "voice" -> {
                voiceFlow.sendVoiceMessage()          // 复用已有语音发送
                "已给${pending.target}发语音"
            }
            "video" -> {
                videoCallFlow.makeCall(useVoice = true) // 复用已有通话发起
                "已发起和${pending.target}的语音通话"
            }
            else -> "不支持的操作"
        }
    } catch (e: Exception) {
        Log.e(TAG, "执行失败", e)
        "执行失败，请重试"
    }
}
```

> 此处 `voiceFlow` 和 `videoCallFlow` 就是之前实现的 `WeChatVoiceFlow` 和 `WeChatVideoCallFlow` 的实例，完全复用。

### 2. 关键词直接执行也走相同路径

```kotlin
// 关键词匹配时直接调用的辅助方法
private suspend fun sendVoiceTo(name: String, durationMs: Long) {
    val ok = chatNavGuard.ensureChatWithTarget(name)
    if (ok) {
        voiceFlow.sendVoiceMessage(durationMs)   // 复用
    }
}

private suspend fun makeVideoCall(useVoice: Boolean, name: String) {
    val ok = chatNavGuard.ensureChatWithTarget(name)
    if (ok) {
        videoCallFlow.makeCall(useVoice)         // 复用
    }
}
```

> 关键词匹配时直接传入从输入中提取的 target（如“给儿子发语音”提取“儿子”）。

### 3. `ChatNavigationGuard.ensureChatWithTarget()` 内部复用搜索 Flow

已在之前实现中调用 `searchFlow.execute(targetName)`，确认无误。

## 二、悬浮窗“搜索联系人”按钮也能复用

如果希望悬浮窗的“搜索联系人”按钮也走这个统一路径，可以将其点击事件改为调用 `chatNavGuard.ensureChatWithTarget(name)`，而非直接 `searchFlow.execute`。这样也能享受“自动开微信+回首页”的好处。但原有独立按钮仍然可用，取决于你的选择。

## 四、代码改动总结

仅需修改两个文件的少量内容：

1. **`IntentHandler.kt`**  
   - `executeConfirmed` 中先调 `ensureChatWithTarget`，再调对应 Flow。  
   - 关键词动作辅助方法（如 `sendVoiceTo`）同样先导航再调 Flow。  
   - （可选）从用户输入中提取联系人的逻辑保持不变。

2. **`ChatNavigationGuard.kt`**  
   - 加入 `WeChatLauncher.launchWeChat` 和前台检测，确保在微信内。  
   - 修改构造函数接受 `Context` 和 `isWeChatForeground` 回调（如之前指南）。  
   - `ensureChatWithTarget` 起始处增加微信前台判断。

其余文件（`WeChatVoiceFlow`、`WeChatVideoCallFlow`、`WeChatSearchContactFlow` 等）**完全不动**。

这样，整个意图识别体系就建立在了已有稳定功能之上，没有任何重复实现。
