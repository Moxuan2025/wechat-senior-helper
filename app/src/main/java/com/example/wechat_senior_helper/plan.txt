问题出在 `isVoiceModeActive()` 中的 OCR 识别可能没拿到正确的底部文本，导致始终返回 false，从而每次都执行切换。

**快速解决**：
1. 在识别前加 `delay(300)` 确保动画完成。
2. 匹配时用更宽松的条件：`contains("按住")` 或 `contains("说话")`，避免空格/换行导致失败。
3. 添加日志打印识别到的底部文字，便于验证。

---

### 修改后的代码（`WeChatVoiceFlow.kt`）

```kotlin
private suspend fun ensureVoiceMode() {
    // 先等一下，避免界面正在切换中
    delay(300)
    if (isVoiceModeActive()) return
    Log.e(TAG, "点击语音图标切换模式")
    input.tapByRatio(VOICE_ICON_X_RATIO, VOICE_ICON_Y_RATIO)
    delay(500)
}

private suspend fun isVoiceModeActive(): Boolean {
    val fullBitmap = screenshotProvider.capture()
    val cropTop = (height * 0.92f).toInt()
    val bottomBitmap = Bitmap.createBitmap(fullBitmap, 0, cropTop, width, height - cropTop)

    val result = ocrEngine.recognize(bottomBitmap, preferChinese = true)
    val text = result.text
    Log.e(TAG, "底部OCR识别文本: [$text]")

    bottomBitmap.recycle()
    fullBitmap.recycle()

    // 宽松匹配，避免识别多空格/换行干扰
    return text.contains("按住") || text.contains("说话")
}
```

**改动要点**：
- 进入 `ensureVoiceMode` 先延迟 300ms，避免界面元素还没渲染完就截图。
- 识别后用 `contains("按住")` 或 `contains("说话")` 双重判断，而不是精确“按住说话”。
- 添加日志 `底部OCR识别文本: [...]`，运行时查看是否真的识别到了正确文字，用于微调裁剪区域或切换坐标。

如果日志显示文字为空或不是底部内容，则调整 `cropTop` 比例（例如 `0.90f`），或改用全屏截图裁剪底部固定的像素高度（如 `height - 200`）。先用日志确认问题，再微调参数。
